Inventory Customer Management GUI

Khoi Ngo

Our website is called Mates & Posts. Users can create, like and comment on posts. Users who are not logged in can still view and search for posts, but they cannot create or like posts. Figure 1 shows the main feed of posts. Users can sort posts by Most Recent, Views and Title. Trending Now showcases the top five posts with the most likes. Recently Viewed keeps track of a user’s previously viewed posts, up to five posts. Users can search for posts by category and title.

Clicking on a post will show it. Administrators can delete any post or comment, logged in users can comment on and like the post, and post authors can edit and delete their posts as shown in Figure 2. After a user likes the post, the user can unlike the post at any time.

Users login with an email and password as shown in Figure 3. If a user forgets their password, they can click “Forgot Password?” to be redirected to the forgot password page in Figure 4. Users can register an account by clicking the Sign Up link where they will be directed to the Registration Page shown in Figure 5.

Logged in users can create new posts, separating categories by a semicolon. Posts cannot exceed 250 characters.

Users can view their profile as shown in Figure 7. The user can see their email, birthdate, and when their account was created. Any posts the user created will be displayed underneath this information. Logged in users can view another user’s profile by clicking on the post author link.

The user can edit any of the information in the form in Figure 8. Only PNG and JGP are accepted files for updating a profile picture. If a user wishes to update their password, they must add their old password to confirm their new password. A user does not need to fill the entire form. Only fields they change will be updated. 

Administrators have an “Administration” tab on the top navigation bar. One of the links under this tab is the index of all registered users as shown in Figure 9. All registered users that are registered are displayed. Administrators to enable or disable users. A minus sign means they will disable and a plus sign means they will enable. Administrators can also delete users. Clicking the user’s name redirects to the user’s profile. Administrators may search for users by name, email, and post titles. After searching, if an admin wants to view all users, simply put “select type” and press submit.

The other link that administrators can access is the analytics page in Figure 10. Administrators can see the number of registered users, the number of active users, the top 5 categories, a pie chart comparing enabled users and disabled users, and a column chart of new users per month.

Summary of Features

  • User Features
    • Browse items without registering
    • Responsive design philosophy (minimum requirements for different non-mobile display sizes) 
    • Server-side security and validation - best practices for password handling
    • Search for items/posts by keyword without registering
    • Users, posts, comments, and likes stored in the database
    • Form validation with JavaScript
    • Register at the site by providing their name, e-mail, and birthdate
    • Allow user login by providing email and password
    • Create and comment on posts when logged into the site
    • Users are required to be able to view/edit their profile
    • User password recovery (via email)
    • Error handling/bad navigation
    • The site must maintain state (user state being logged on)
  • Admin
    • Search for a user by name, email or post
    • Enable/disable/delete users
    • Remove posts and comments (only post authors can edit their post)
  • Additional
    • Search and analysis of topics
    • Hot threads/hot item tracking
    • Visual display of updates, etc (site usage charts, etc) for admins
    • Alerts on-page changes
    • Styling flourishes
    • All published posts by the user shown in their profile
    • Can asynchronously like/unlike a post
    • Sort posts by date, views or title
    • Each post can have multiple searchable categories
    • Tracks recently viewed posts with/without logging in
